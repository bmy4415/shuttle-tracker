## 💡 향후 기능 아이디어

### 1. 탑승 여부 관리 시스템 (다음 Phase 예정)

#### 사용자 그룹 시스템
- **Firebase Anonymous Auth 기반 자동 회원가입**
  - 기기별 자동 회원가입 (가입 절차 최소화)
  - 앱 삭제 후 재설치시 새로운 사용자로 취급 (시즌성 특성상 적합)
  - 랜덤 닉네임 자동 생성 (예: "야생의 코끼리", "용감한 사자")
  - 첫 실행시 닉네임 설정 팝업 제공

#### 그룹 생성 및 참여 시스템
- **버스기사가 그룹 생성**: 고유 그룹 코드 자동 생성 (예: "LION-3A-7B2K")
- **학부모/선생님 그룹 참여**: 그룹 코드 입력으로 가입
- **역할 선택**: driver(기사) / teacher(선생님) / parent(학부모)
- **그룹 내 권한**: 기사와 선생님은 동일한 읽기/쓰기 권한, 학부모는 자녀 관련만

#### 탑승 여부 설정 기능
- **기본값**: 모든 학생 "탑승 예정" 상태
- **학부모 기능**: "오늘은 탑승하지 않아요" 버튼
  - 확인 팝업: "오늘 셔틀버스를 이용하지 않으시겠어요?"
  - 취소 가능: "탑승 예정으로 변경하기" 버튼 제공
- **실시간 알림**: 탑승 안함 설정시 그룹 내 기사/선생님에게 자동 알림
- **시간 제한 없음**: 버스 출발 이전 언제든 변경 가능

#### 기사/선생님 화면
- **학생 목록**: 탑승 예정/미탑승 상태 실시간 확인
- **시각적 구분**:
  - 🚌 탑승 예정 (기본, 밝은 색상)
  - ❌ 오늘은 탑승하지 않음 (dimmed 처리)
- **동일 권한**: 기사와 선생님 모두 학생 목록 관리 가능

### 2. 방 기반 실시간 위치 공유 시스템 (미래 구현 예정)
완전한 서버 기반 실시간 위치 공유 시스템으로, 다음과 같은 플로우를 제공:

#### 기본 플로우
1. **방 생성**: 버스기사가 방을 만든다 (고유한 코드 생성)
2. **방 참여**: 학부모는 코드를 입력하여 버스기사 방에 참여
3. **참여자 관리**: 버스기사는 방에 참여한 학부모 목록을 실시간으로 확인

#### 운행 관리 시스템
4. **운행 준비**: 버스기사가 운행 시작 버튼을 누르면 참여한 학부모 목록 표시
   - **접속한 학부모**: 밝은색으로 표시 (위치 동기화 중)
   - **접속하지 않은 학부모**: dimmed 처리로 표시
   - 실시간 접속 상태 모니터링

5. **학부모 접속**: 학부모가 코드로 버스기사 방 참여 가능
6. **위치 공유 시작**: 버스기사 운행 시작 → 본인 위치 서버 공유 시작

#### 학부모 화면 분기
7. **운행 중**: 버스기사가 운행 시작한 경우
   - 지도 화면으로 이동
   - 본인 위치 + 버스기사 위치 실시간 확인
   - 자동 위치 동기화

8. **운행 대기**: 버스기사가 운행 시작하지 않은 경우
   - "버스기사가 운행을 시작하지 않았습니다." 안내 화면 표시
   - 운행 시작 대기 상태

#### 기술적 특징
- `isWaitingForPickup` 상태를 활용한 학부모 접속 상태 관리
- 서버 기반 실시간 위치 동기화
- 방 코드 기반 그룹 관리 시스템

### 탑승 관리 시스템 (미래 구현 예정)
- **기사 기능**: 탑승 버튼으로 자녀 탑승 상태 관리
  - 각 학부모별 탑승/하차 버튼
  - 탑승 완료 시 해당 자녀 상태 업데이트
- **학부모 기능**: 자녀 탑승 후 지도 뷰 변경
  - 자녀가 탑승하면 기사 위치를 메인으로 표시
  - "자녀가 버스에 탑승 중" 상태 표시
  - 기사 위치 실시간 추적으로 전환
